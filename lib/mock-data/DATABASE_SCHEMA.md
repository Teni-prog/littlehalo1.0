## Core User Tables

### `users` (Authentication)

**Purpose:** Base authentication table (managed by Supabase Auth)

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| id | uuid | PRIMARY KEY | Auto-generated by Supabase Auth |
| email | varchar(255) | UNIQUE, NOT NULL | User's email address |
| email_verified | boolean | DEFAULT false | Email verification status |
| phone | varchar(20) | UNIQUE | Phone number |
| phone_verified | boolean | DEFAULT false | Phone verification status |
| user_type | enum | NOT NULL | 'parent', 'sitter', or 'admin' |
| created_at | timestamp | DEFAULT NOW() | Account creation date |
| updated_at | timestamp | DEFAULT NOW() | Last update timestamp |
| last_login | timestamp | | Last login timestamp |
| is_active | boolean | DEFAULT true | Account active status |
| avatar_url | text | | Profile picture URL |

**Indexes:**

- `idx_users_email` on email
- `idx_users_user_type` on user_type
- `idx_users_created_at` on created_at

## Booking Request Tables

### `booking_requests`

**Purpose:** Pending booking requests from parents to sitters (before confirmation)

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| id | uuid | PRIMARY KEY | Auto-generated |
| parent_id | uuid | NOT NULL, FK | References users.id |
| sitter_id | uuid | NOT NULL, FK | References users.id |
| job_id | uuid | FK | References jobs.id (if from job posting) |
| status | enum | NOT NULL, DEFAULT 'pending' | 'pending', 'accepted', 'declined', 'expired', 'cancelled' |
| service_date | date | NOT NULL | Date of service |
| start_time | time | NOT NULL | Start time |
| end_time | time | NOT NULL | End time |
| duration_hours | decimal(4,2) | NOT NULL | Total duration in hours |
| hourly_rate | decimal(10,2) | NOT NULL | Proposed hourly rate |
| total_pay | decimal(10,2) | NOT NULL | Total proposed payment |
| children_count | integer | NOT NULL | Number of children |
| children_ids | jsonb | DEFAULT '[]' | Array of child IDs |
| special_requests | text | | Special instructions |
| city | varchar(100) | | Service location city |
| full_address | text | | Complete address (shown after acceptance) |
| decline_reason | text | | Reason if declined |
| declined_at | timestamp | | When request was declined |
| accepted_at | timestamp | | When request was accepted |
| expires_at | timestamp | | Auto-expiration time |
| booking_id | uuid | FK | References bookings.id (created on acceptance) |
| created_at | timestamp | DEFAULT NOW() | Request creation date |
| updated_at | timestamp | DEFAULT NOW() | Last update timestamp |

**Indexes:**

- `idx_breq_parent_id` on parent_id
- `idx_breq_sitter_id` on sitter_id
- `idx_breq_status` on status
- `idx_breq_service_date` on service_date
- `idx_breq_created_at` on created_at
- `idx_breq_expires_at` on expires_at

**Notes:**

- When a parent books a sitter (directly or via job application acceptance), a booking_request is created with status 'pending'
- Sitter can Accept (creates a confirmed booking + links booking_id) or Decline (with optional reason)
- Requests auto-expire after a configurable time window
- See MISSING-PAGES-AUDIT.md for the full workflow

## Supabase Considerations

### Row Level Security (RLS) Policies

#### Users & Profiles

```sql
-- Users can read their own profile
CREATE POLICY "Users can view own profile"
  ON profiles FOR SELECT
  USING (auth.uid() = id);

-- Users can update their own profile
CREATE POLICY "Users can update own profile"
  ON profiles FOR UPDATE
  USING (auth.uid() = id);
```

#### Bookings

```sql
-- Parents can view their bookings
CREATE POLICY "Parents can view own bookings"
  ON bookings FOR SELECT
  USING (auth.uid() = parent_id);

-- Sitters can view their bookings
CREATE POLICY "Sitters can view assigned bookings"
  ON bookings FOR SELECT
  USING (auth.uid() = sitter_id);
```

#### Messages

```sql
-- Users can view messages in their conversations
CREATE POLICY "Users can view own messages"
  ON messages FOR SELECT
  USING (
    auth.uid() = sender_id OR 
    auth.uid() = receiver_id
  );
```

CREATE TRIGGER update_updated_at
  BEFORE UPDATE ON <table>
